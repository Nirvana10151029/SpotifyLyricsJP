# Changelog

## 2.0.0 — 2026-07-20

- Spotify公式サイト版とSpicetifyに対応し、別ウィンドウからSpotify右側の内蔵パネルへ変更
- 再生曲の自動検出、同期ハイライト、自動スクロールをSpicetify Player APIへ移植
- LRCLIBで通常歌詞しか見つからない場合も検索を続け、別候補またはLyricaの同期歌詞を自動優先
- LRCLIB、Lyrica経由のYouTube Music・NetEase・Megalobiz・Musixmatch・SimpMusic、Lyrics.ovhに対応
- 曲名・アーティスト・アルバム・再生時間の照合と、45秒以上異なる候補の除外を移植
- 「再取得」と「別ソース」をSpotify内蔵パネルへ追加
- 取得失敗後は自動再試行せず、曲変更または手動操作まで停止するよう実装
- 無料翻訳、Gemini、DeepL、GPTの切り替えとAPI設定画面を移植
- Gemini・GPTは行ID付きJSONを検証し、歌詞行の結合・省略による同期ずれを防止
- DeepL API Free/Proのエンドポイントをキーから自動判定
- GPT自然訳をGPT-5.6 SolとResponses APIの構造化出力に更新
- AI翻訳失敗時の無料翻訳フォールバックを実装
- Web版Spotify・Spicetify・拡張機能を確認して導入するWindows用インストーラーを追加
- 拡張機能だけを安全に取り外すアンインストーラーを追加

## 1.4.1 — 2026-07-19

- LRCLIBで同期情報のない通常歌詞が見つかっても、すぐには確定せず同期歌詞の検索を続けるよう変更
- LRCLIB内の別候補とLyrica経由の他サービスを検索し、正しく照合できた同期歌詞があれば自動で入れ替え
- 同期歌詞がどのサービスにもない場合だけ、保留していた通常歌詞を表示するよう優先順位を改善
- 同期情報がない場合のステータス欄に、実際の歌詞ソース名を表示

## 1.4.0 — 2026-07-19

- 翻訳方式にDeepLとGPT自然訳を追加し、無料翻訳・Gemini・DeepL・GPTの4種類から選択可能に変更
- DeepL API Freeキー（末尾 `:fx`）とProキーを自動判定し、対応する公式エンドポイントへ接続
- DeepLへ曲名・アーティスト・前後の歌詞を文脈として渡し、1行ごとの対応を維持して翻訳
- GPT自然訳にGPT-5.6 TerraとResponses APIの構造化出力を採用
- GPTの出力をID付きJSONで検証し、歌詞の結合・省略による同期ずれを防止
- Gemini・DeepL・OpenAIのAPIキーを個別にWindows暗号化して保存
- 「AI設定」を「API設定」へ変更し、現在選択中のサービスのキーを設定できるよう改善
- 各サービスが失敗した場合は、その曲の残りを無料翻訳へ自動フォールバック

## 1.3.1 — 2026-07-19

- AI自然訳の選択時に設定画面の所有者変数 `$form` がイベントスコープで見つからず、例外画面が出る問題を修正
- メイン画面をアプリ全体の変数として保持し、AI設定と「常に手前」から安全に参照するよう変更
- 翻訳方式・AI設定のイベントで例外を捕捉し、Windowsの未処理例外画面を防止

## 1.3.0 — 2026-07-19

- 画面から「無料翻訳」と「AI自然訳（Gemini）」を切り替える機能を追加
- AI自然訳では前後最大24行をまとめて翻訳し、比喩・口語・文脈を踏まえた日本語を生成
- 構造化JSONで原文と訳文を1行ずつ対応させ、同期タイミングのずれを防止
- Gemini APIキーをアプリ内で入力・変更できる「AI設定」を追加
- APIキーをWindowsのユーザー単位暗号化（DPAPI）でローカル保存し、ログには記録しない設計に変更
- Geminiの通信失敗や出力不整合時は無料翻訳へ自動フォールバック
- 使用中の翻訳方式をステータス欄に表示

## 1.2.0 — 2026-07-19

- LRCLIB以外の同期歌詞をLyrica経由で取得する機能を追加
- YouTube Music、NetEase、Megalobiz、Musixmatch、SimpMusicを同期歌詞の候補として追加
- 通常検索はLRCLIBの次にLyricaの同期歌詞を検索し、最後にLyrics.ovhの通常歌詞へフォールバック
- 「別ソース」ボタンではLRCLIBを除外し、Lyricaの同期歌詞を優先
- LyricaがLRCLIB由来の結果を返した場合も除外し、別サービス由来だけを採用
- Lyricaの結果にも曲名・再生時間・アルバムの誤取得防止判定を適用
- 実際の取得元を `Lyrica/YouTube Music` などの形式でステータス欄に表示

## 1.1.1 — 2026-07-19

- 同名の別楽曲を選ぶことがある問題を修正
- 候補選択に再生時間とアルバムの一致判定を追加し、45秒以上異なる候補を除外
- `Remastered` などの版表記を除いた曲名で照合しつつ、再生時間で正しい版を判定
- アーティスト表記が一致しない候補は、再生時間またはアルバムも一致する場合だけ採用
- 誤った歌詞が出た場合にLyrics.ovhへ切り替えられる「別ソース」ボタンを追加

## 1.1.0 — 2026-07-19

- LRCLIBで見つからない場合にLyrics.ovhを第二の歌詞ソースとして検索する機能を追加
- Spotifyが「Eminem」を「エミネム」のようにローカライズして渡す場合に備え、LRCLIBを曲名だけでも再検索するよう改善
- Lyrics.ovhの候補検索から英字の正式アーティスト名を補完する処理を追加
- 取得元がLRCLIBかLyrics.ovhかをステータス欄に表示
- 診断ツールでLRCLIBとLyrics.ovhを個別に接続確認できるよう変更

## 1.0.5 — 2026-07-19

- 歌詞が0件のとき、空の検索結果に対する `Count` 参照で取得エラーになる問題を修正
- 取得エラー後に同じ曲を0.85秒間隔で再試行し続ける無限ループを修正
- 取得できなかった曲は自動再試行せず、「再取得」を押したときだけ再試行するよう変更
- Windows PowerShell 5.1で翻訳サービスのJSON配列が単一要素に展開された場合も処理できるよう改善

## 1.0.4 — 2026-07-19

## 1.0.3 — 2026-07-19

- 動作確認中はコンソールを表示し、隠れたエラーを確認できるよう変更

## 1.0.1 — 2026-07-19

- Windows PowerShell 5.1でWinRTの非同期処理を待てない問題を修正
